## Decisions
- **Orphaned dead component was the actual bootstrap target.** `src/main.ts` imported `AppComponent` from `./app/app.component` — a leftover file with an old inline 2-link nav template (matching the unused `.app-header`/`.app-main` CSS classes noticed in `styles.sass` back in the first session). Meanwhile `src/app/app.ts` (`export class App`, Angular's current no-`.component.`-infix naming) is the file that actually gets edited by tooling/scaffolding conventions and the one I was redesigning all along — but it was never wired up. Confirmed via `grep` that nothing else referenced `app.component.ts`, deleted it, and pointed `main.ts` at the real `App` class. This is why nav/hero/footer looked "half-styled" for several iterations even after clearing `.angular/cache` and fully restarting the dev server — I was redesigning a file that was never being bootstrapped.
- **Zoneless Angular + plain mutable properties = silently broken async UI.** This app has no `zone.js` dependency at all (confirmed via `package.json`/`angular.json` — it's a zoneless app by default, matching current Angular scaffolding). Component state mutated inside an `async`/`await` continuation (e.g. `this.clauses = await this.api.getClauses(...)`) never triggers a re-render in zoneless mode, because the continuation resumes as a bare microtask with nothing to notify Angular's change detector — even though the method itself was invoked from a template `(click)` handler. This silently broke all three interactive pages: the API calls succeeded (confirmed 200 responses with correct data via Playwright network inspection) but the DOM never updated. Fixed by converting all async-mutated state (`result`, `loading`, `error`, `clauses`, `file`, `dragging`, `message`, `extracting`) to Angular `signal()`s across `ComparatorComponent`, `LeaseUploadComponent`, `ClausesComponent`, with templates updated to invoke them as functions (`result()`, `@if (result(); as result)`, etc.). Left `[(ngModel)]`-bound primitives (`suburb`, `postcode`, `leaseId`) as plain properties since those are driven by native DOM input events, which zoneless CD does pick up.
- **Frontend/backend dev port mismatch.** `environment.ts` had `apiUrl: 'http://localhost:5000'` (leftover default from before this session's backend audit), but `backend/Properties/launchSettings.json` (added earlier today) defaults to port 5050. Updated `environment.ts` to `5050` and fixed the stale "(default 5000)" mention in `backend/README.md`.
- **Angular template whitespace trimming ate a literal space.** `<span>Know your</span> <span>rent.</span>` rendered as "yourrent." with no gap — Angular's default `preserveWhitespaces: false` strips whitespace-only text nodes between elements, including ones that are visually significant for adjacent `inline-block` spans. Fixed with `margin-right: 0.24em` on `.hero-title span` in CSS instead of relying on template whitespace.
- **Confirmed:** `chromium-cli` (the `/run` skill's preferred browser driver) isn't installed in this environment; fell back to a hand-rolled Playwright script per the skill's documented fallback path, installed into the scratchpad dir (not the project) to avoid adding a project dependency for a one-off verification.
- Following the root `CLAUDE.md` "Important" directives from here on: starting/stopping dev servers is handled autonomously (confirmed today it's possible to `tasklist`/`taskkill` and restart processes the user started in their own terminal, since it's the same OS process table), and only interrupting the user when genuinely blocked rather than for routine confirmations.
